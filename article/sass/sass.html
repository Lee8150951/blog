    Sass笔记

    Oreo! 2021/3/25 SassCSS预处理前端

    # 一、概述

    # 简介

    Sass是一种CSS的拓展工具，在CSS原有的基础上添加了以下功能：

    1、变量（variables）

    2、嵌套（nested rules）

    3、混合（mixins）

    4、导入（inline imports）

    ...

    使用目的：更好的组织管理样式文件，更加高效的开发项目

    特色：

    • 完全兼容CSS3语法
    • 通过函数进行颜色值与属性值的运算
    • 提供控制指令等高级功能
    • 自定义输出格式

    # 语法格式

    Sass拥有两套语法格式，首先是SCSS语法格式，这是对CSS语法的一种拓展，所有的CSS语法在SCSS中都是支持的，使用.scss作为拓展名。另外一种是SASS语法格式，使用缩进格式（在这一方面更像Less），它使用缩进格式代替花括号表示属性属于某一个选择器，使用.sass作为拓展名

    其实在SASS中可以使用任意一种语法格式进行书写，通过导入（@import）到另外一种格式中进行使用，或者通过sass-convert命令行工具进行转换

    # Convert Sass to SCSS
$ sass-convert style.sass style.scss

# Convert SCSS to Sass
$ sass-convert style.scss style.sass

    # 二、嵌套语法

    Sass可以将一套CSS样式嵌入另一套样式中，内层样式将外层样式作为父选择器

    # 一般使用

    一般情况下，CSS如果需要父选择器要写成如下情况

    .nav {
    height: 30px;
}
.nav ul {
    margin: 0;
}
.nav ul li {
    float: left;
    list-style: none;
    padding: 5px;
}

    但是这种写法非常冗余，.nav这个类选择器出现了多次，使用scss语法可以有效地避免这种情况的发生

    // 以上可以写为以下形式
.nav {
    height: 30px;
    ul {
        margin: 0;
        li {
            float: left;
            list-style: none;
            padding: 5px;
        }
    }
}

    # 选择器名拼接

    如果出现了需要使用:hover的类选择器，则在scss中需要按照下面的方式书写：

    #main {
  color: black;
  a {
    font-weight: bold;
    &:hover { color: red; }
  }
}

    & 必须作为选择器的第一个字符，其后可以跟随后缀生成复合的选择器

    可以理解成&起到的是拼接的作用，如a {&:hover {}}编译出来的效果是a :hover{}，甚至在普通的类选择器中都可以使用，比如sidebar {&-title {}}，编译出来的效果是sidebar-title {}

    # 样式嵌套

    如果出现了一系列针对同一属性的修改，如：font，background，padding等等样式属性时，可以使用嵌套的方式对原本冗余的css代码进行改造，如：

    body {
    font-size: large;
    font-weight: 100;
    font-family: Arial, Helvetica, sans-serif;
}

    可以改造成如下的情况：

    body {
    font: {
        family: Arial, Helvetica, sans-serif;
        weight: 100;
        size: large;
    }
}

    # 三、SassScript

    # Interactive Shell

    在命令行中输入 sass -i，然后输入想要测试的 SassScript 查看输出结果，可以对输入的SassScript进行测试

    # 变量($)

    使用$可以对css属性进行赋值，赋值方式和普通的css语法是相同的

    $length: 10px

    在使用时直接调用即可：

    #main {
  width: $length;
}

    变量支持块级作用域，嵌套规则内定义的变量只能在嵌套规则内使用（局部变量），不在嵌套规则内定义的变量则可在任何地方使用（全局变量）。将局部变量转换为全局变量可以添加 !global 声明：

    #main {
  $width: 5em !global;
  width: $width;
}

#sidebar {
  width: $width;
}

    Sass中的变量支持六种数据类型：数字、字符串、颜色、Boolean、null、数组、maps

    Tips：maps相当于JavaScript中的对象类型，由许多键值对构成(key1: value1, key2: value2)

    # 运算

    SassScript 可以支持数字的加减乘除、取整等运算 (+, -, *, /, %)，如果必要会在不同单位间转换值，如：

    p {
  width: 1in + 8pt;
}

    编译为

    p {
  width: 1.111in; }

    SassScript 还支持颜色值 的运算，颜色值的运算是分红绿蓝进行分别计算的

    Tips：css中的RGB颜色值是六位十六进制构成的，每两位构成一组颜色，一二位构成红色系，三四位构成绿色系，五六位构成蓝色系，如：#010203中01为红色系，02为绿色系，03为蓝色系

    所以SassScript的颜色值计算就是根据不同的色系进行计算的，如下：

    p {
  color: #010203 + #040506;
}

    得到的结果就是：

    p {
  color: #050709; }

    # 插值语句(#{})

    通过 #{} 插值语句可以在选择器或属性名中使用变量：

    $name: foo;
$attr: border;
p.#{$name} {
  #{$attr}-color: blue;
}

    编译出来的结果就是：

    p.foo {
  border-color: blue; }

    # 四、@-Rules 与指令

    # @import

    Sass 拓展了 @import 的功能，允许其导入 SCSS 或 Sass 文件。被导入的文件将合并编译到同一个 CSS 文件中

    被导入的文件中所包含的变量或者混合指令 (mixin) 都可以在导入的文件中使用。

    如果存在需要导入scss文件但是又不希望其进行编译的，只需要在文件名前面加上_，但是在引入时注意不需要_，这个操作方式称为分音（Partials）

    # @extend

    使用@extend可以让一个选择器的样式完全继承另一个选择器的样式，并在原本的基础上进行拓展

    .error {
  border: 1px #f00;
  background-color: #fdd;
}
.seriousError {
  @extend .error;
  border-width: 3px;
}

    这样的情况下，不再需要在html标签类中写.error和.seriousError两个类，只需要使用.seriousError即可

    @extend存在继续延伸的属性，当一个选择器延伸给第二个后，可以继续将第二个选择器延伸给第三个

    # 五、控制指令

    # @if

    当 @if 的表达式返回值不是 false 或者 null 时，条件成立，输出 {} 内的代码

    p {
  @if 1 + 1 == 2 { border: 1px solid; }
  @if 5 < 3 { border: 2px dotted; }
  @if null  { border: 3px double; }
}

    编译为：

    p {
  border: 1px solid; }

    @if 声明后面可以跟多个 @else if 声明，或者一个 @else 声明。如果 @if 声明失败，Sass 将逐条执行 @else if 声明，如果全部失败，最后执行 @else 声明，例如：

    $type: monster;
p {
  @if $type == ocean {
    color: blue;
  } @else if $type == matador {
    color: red;
  } @else if $type == monster {
    color: green;
  } @else {
    color: black;
  }
}

    编译为

    p {
  color: green; }

    # @for

    @for 指令可以在限制的范围内重复输出格式，每次按要求（变量的值）对输出结果做出变动。这个指令包含两种格式：@for $var from <start> through <end>，或者 @for $var from <start> to <end>，区别在于 through 与 to 的含义：当使用 through 时，条件范围包含 <start> 与 <end> 的值，而使用 to 时条件范围只包含 <start> 的值不包含 <end> 的值。另外，$var 可以是任何变量，比如 $i；<start> 和 <end> 必须是整数值。

    @for $i from 1 through 3 {
  .item-#{$i} { width: 2em * $i; }
}

    编译为：

    .item-1 {
  width: 2em; }
.item-2 {
  width: 4em; }
.item-3 {
  width: 6em; }

    # @each

    @each 指令的格式是 $var in <list>, $var 可以是任何变量名，比如 $length 或者 $name，而 <list> 是一连串的值，也就是值列表。

    @each 将变量 $var 作用于值列表中的每一个项目，然后输出结果，例如：

    @each $animal in puma, sea-slug, egret, salamander {
  .#{$animal}-icon {
    background-image: url('/images/#{$animal}.png');
  }
}

    编译为：

    .puma-icon {
  background-image: url('/images/puma.png'); }
.sea-slug-icon {
  background-image: url('/images/sea-slug.png'); }
.egret-icon {
  background-image: url('/images/egret.png'); }
.salamander-icon {
  background-image: url('/images/salamander.png'); }

    # 六、混合（Mixin）

    混合指令（Mixin）用于定义可重复使用的样式，避免了使用无语意的 class，比如 .float-left。混合指令可以包含所有的 CSS 规则，绝大部分 Sass 规则，甚至通过参数功能引入变量，输出多样化的样式。

    简单来说：

    • @mixin 指令允许我们定义一个可以在整个样式表中重复使用的样式。

    • @include 指令可以将混入（mixin）引入到文档中。

    @mixin alert {
    color: aqua;
    background-color: #fff;
    a {
        color: #000;
    }
}

.alert-warning {
    @include alert();
}

    另外，在@Mixin中可以使用传参的方式（类似函数），使样式能够更加具有复用性：

    @mixin sexy-border($color, $width: 1in) {
  border: {
    color: $color;
    width: $width;
    style: dashed;
  }
}
p { @include sexy-border(blue); }
h1 { @include sexy-border(blue, 2in); }

    # 七、输出格式

    # :nested

    Nested （嵌套）样式是 Sass 默认的输出格式，能够清晰反映 CSS 与 HTML 的结构关系。选择器与属性等单独占用一行，缩进量与 Sass 文件中一致，每行的缩进量反映了其在嵌套规则内的层数。当阅读大型 CSS 文件时，这种样式可以很容易地分析文件的主要结构。

    # :expanded

    Expanded 输出更像是手写的样式，选择器、属性等各占用一行，属性根据选择器缩进，而选择器不做任何缩进。

    # :compact

    Compact 输出方式比起上面两种占用的空间更少，每条 CSS 规则只占一行，包含其下的所有属性。嵌套过的选择器在输出时没有空行，不嵌套的选择器会输出空白行作为分隔符。

    # :compressed

    Compressed 输出方式删除所有无意义的空格、空白行、以及注释，力求将文件体积压缩到最小，同时也会做出其他调整，比如会自动替换占用空间最小的颜色表达方式。

    简单来说就是：

    # 另外，输出的css有四种格式，对应了不同的输出指令
# 1、nested嵌套（默认输出格式）：sass [源文件]:[输出文件]
# 2、compact紧凑：sass [源文件]:[输出文件] --style compact
# 3、expanded拓展：sass [源文件]:[输出文件] --style expanded
# 4、compressed压缩：sass [源文件]:[输出文件] --style compressed
