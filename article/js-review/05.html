    JavaScript复习-事件与事件流

    Oreo! 2022/7/11 JavaScript

    # 一、事件

    事件使用户操作网页时发生的交互动作，比如：click、move等等，事件除了用户触发之外，也可以是文档加载、窗口滚动、大小调整等等。每一个事件都封装成一个event对象，包含该对象发生时的所有信息（event对象的属性），对事件进行的操作（event对象的方法）

    所有的event对象，例如：MouseEvent、KeyBoardEvent对象都继承自一个名为Event的类

    # 1.1Event对象

    # 1.1.1常用属性

    • 事件定位相关属性

    在MouseEvent对象里的属性，有很多带X/Y的属性，它们都和事件的位置相关。具体包括：x/y、clientX/clientY、pageX/pageY、screenX/screenY、layerX/layerY、offsetX/offset 六对

    1、x/y与clientX/clientY值一样，表示距浏览器可视区域（工具栏除外区域）左/上的距离

    2、pageX/pageY，距页面左/上的距离，它与clientX/clientY的区别是不随滚动条的位置变化

    3、screenX/screenY，距计算机显示器左/上的距离，拖动你的浏览器窗口位置可以看到变化

    4、layerX/layerY与offsetX/offsetY值一样，表示距有定位属性的父元素左/上的距离

    • 其他属性

    1、target：发生事件的节点

    2、currentTarget：当前正在处理的事件的节点，在事件捕获或冒泡阶段

    3、timeStamp：事件发生的时间，时间戳

    4、bubbles：事件是否冒泡

    5、cancelable：事件是否可以用preventDefault()方法来取消默认的动作

    6、keyCode：按下的键的值

    # 1.1.2常用方法

    • event.preventDefault()阻止元素默认的行为，如链接的跳转、表单的提交

    • event.stopPropagation()阻止事件冒泡

    • event.initEvent()初始化新事件对象的属性，自定义事件会用，不常用

    • event.stopImmediatePropagation()可以阻止掉同一事件的其他优先级较低的侦听器的处理

    # 1.2事件的三大模型

    # 1.2.1 DOM0级事件模型

    在DOM0级事件模型中，事件发生后没有传播的概念，没有事件流。事件发生后立即处理。监听函数只是元素的一个属性值，通过指定元素的属性值来绑定监听器。

    <!-- 方法一 -->
<input type=”button” onclick=”func1()”/>
<script>
  funcs() {...}
</script>

<!-- 方法二 -->
<input type=”button”/>
<script>
  document.getElementsByTagName(‘input’)[0].onclick = function() {...}
</script>

    优点：对所有浏览器都兼容

    缺点：逻辑与显示无法分离；无法通过事件的冒泡、委托等机制完成更多事情

    # 1.2.2 IE事件模型

    在该事件模型中一次事件共有两个过程组成：事件处理阶段和事件冒泡阶段

    事件处理会首先执行目标元素绑定的监听事件，然后进入事件冒泡阶段

    • 事件冒泡：事件从目标元素冒泡到document，依次检查经过的节点是否绑定了事件监听函数，有则执行

    这种模型通过attachEvent()方法来添加监听函数，也可以同时添加多个监听函数，会依次执行

    缺点：兼容性，只有IE系列浏览器才可以这样写

    # 1.2.3 DOM2级事件模型

    在该事件模型中，一次事件需要经过三个过程：事件捕获、事件执行、事件冒泡

    • 事件捕获：事件从document一直向下传播到目标元素，依次检查经过的节点是否绑定了事件监听函数，有则执行

    这种事件模型中绑定函数是addEventListener()其中地上那个参数可以指定事件是否在捕获阶段执行

    注：addEventListener第三个参数true表示事件捕获，false为事件冒泡，默认为事件冒泡

    var a = document.getElementById('a');
function func() {
  console.log("OK")
}
if(a.attachEvent){
  // IE事件模型
  a.attachEvent('onclick', func);
}
else{
  // DOM2事件模型
  a.addEventListener('click', func, false);
}

    # 二、事件流

    事件流描述的就是从页面中接收事件的顺序

    而早期的IE和Netscape提出了完全相反的事件流概念，IE事件流是事件冒泡，而Netscape的事件流就是事件捕获

    # 2.1事件捕获

    Netscape的事件流是事件捕获，即从document逐级向下传播到目标元素。由于IE低版本浏览器不支持，所以很少使用事件捕获

    # 2.2事件冒泡

    IE提出的事件流是事件冒泡，即从下至上，从目标触发的元素逐级向上传播，直到window对象

    # 2.3 DOM2整合

    ECMAScript在DOM2中对事件流进行了进一步规范，基本上就是上述二者的结合

    DOM2级事件规定的事件流包括三个阶段：（1）事件捕获阶段 （2）处于目标阶段 （3）事件冒泡阶段

    # 2.4举例

    one.addEventListener('click',function(){alert('one');},true);
two.addEventListener('click',function(){alert('two,bubble');},false);
two.addEventListener('click',function(){alert('two,capture');},true);
three.addEventListener('click',function(){alert('three,bubble');},true);
four.addEventListener('click',function(){alert('four');},true);
    • 如果two为目标元素，目标元素的同类型事件按顺序执行，而其他元素根据W3C的标准执行，即先捕获后冒泡。 点击two执行结果：“one” “two,bubble” “two,capture”
    • 如果目标元素不是two，则two的同类型事件按先捕获后冒泡触发执行，只不过两个事件都绑定在同一个DOM元素上，例如点击three执行结果：“one” “two,capture” “three,bubble” “two,bubble”

    # 三、事件委托

    事件委托的本质是利用了浏览器的事件冒泡机制，因为事件在冒泡的过程中会上传到父节点，并且父节点可以通过事件对象获取到目标节点，因此可以将最节点的监听函数定义在父节点上，由父节点监听函数统一处理多个子节点的事件

    • 优点：使用事件委托可以不必在每一个子元素上都绑定一个监听事件，这样减少了内存消耗。另外事件委托可以实现事件中的动态绑定，新增一个节点不必单独为这个节点添加监听事件

    示例：

    <body>
    <ul id="ul">
        <li id="li">0</li>
        <li>1</li>
        <li>2</li>
        <li>3</li>
    </ul>
    <button id="btn">
        add
    </button>
</body>
<script>
    let ul = document.getElementById("ul")
    ul.addEventListener("click", function(e) {
        ul.removeChild(e.target)
    }, true)
    let btn = document.getElementById("btn")
    btn.addEventListener("click", function() {
        let li = document.createElement("li")
        ul.appendChild(li)
    })
</script>

    # 四、通用事件侦听器

    const EventUtils = {
    // element表示元素，handle为回调函数，type是监听的事件类型
    // 添加监听器
    addEvent: function (element, type, handle) {
        // DOM2事件模型
        if (element.addEventListener) {
            element.addEventListener(type, handle, false)
        }
        // IE事件模型
        else if (element.attachEvent) {
            element.attachEvent("on" + type, handle)
        }
        else {
            element["on" + type] = handle
        }
    }
    // 删除事件
    removeEvent: function (element, type, handle) {
        if (element.removeEventListener) {
            element.removeEventListener(type, handle, false)
        }
        else if (element.detachEvent) {
            element.detachEvent("on" + type, handle)
        }
        else {
            element["on" + type] = null
        }
    }
    // 获取事件目标
    getTarget: function (event) {
        // IE下，event对象有srcElement属性，但是没有target属性
        // Firefox下，event对象有target属性，但是没有srcElement属性
        // 两者作用相当
        return event.target || event.srcElement
    }
    // 获取event对象
    getEvent: function (event) {
        // 在IE/Opera中，用window.event写法
        // 在Firefox里面,  用event写法
        return event || window.event
    }
    // 阻止事件冒泡
    stopPropagation: function () {
        // stoppropagation适用于Firefox等浏览器，不支持IE浏览器。
        // cancelBubble只支持IE浏览器
        if (event.stopPropagation) {
            event.stopPropagation
        } else {
            event.cancelBubble = true
        }
    }
    // 取消默认事件
    preventDefault: function () {
        if (event.preventDefault) {
            event.preventDefault()
        } else {
            // returnValue是IE的一个属性，如果设置了该属性，它的值比事件句柄的返回值优先级要高
            // 值设置为false，可以取消发生事件源元素的默认动作
            event.returnValue = false
        }
    }
}
