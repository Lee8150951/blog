    小程序开发笔记（三）Django模型

    Oreo! 2022/3/18 pythonDjango

    Django 对各种数据库提供了很好的支持，包括：PostgreSQL、MySQL、SQLite、Oracle

    Django 为这些数据库提供了统一的调用API，可以根据自己业务需求选择不同的数据库

    安装对于Python的mysql驱动，可以执行命令：sudo pip3 install pymysql

    # 一、Django ORM

    Django 模型使用自带的 ORM

    对象关系映射（Object Relational Mapping，简称 ORM ）用于实现面向对象编程语言里不同类型系统的数据之间的转换

    ORM 在业务逻辑层和数据库层之间充当了桥梁的作用

    ORM 是通过使用描述对象和数据库之间的映射的元数据，将程序中的对象自动持久化到数据库中

    使用 ORM 的好处：

    • 提高开发效率
    • 不同数据库可以平滑切换

    使用 ORM 的缺点：

    • ORM 代码转换为 SQL 语句时，需要花费一定的时间，执行效率会有所降低
    • 长期写 ORM 代码，会降低编写 SQL 语句的能力

    ORM 解析过程:

    • ORM 会将 Python 代码转成为 SQL 语句
    • SQL 语句通过 pymysql 传送到数据库服务端
    • 在数据库中执行 SQL 语句并将结果返回

    # 二、数据库配置

    在settings.py中设置对MySQL数据库的连接

    DATABASES = {
    'default':
    {
        # 数据库引擎
        'ENGINE': 'django.db.backends.mysql',
        # 数据库名称
        'NAME': 'test',
        # 数据库地址，本机 ip 地址 127.0.0.1
        'HOST': '127.0.0.1',
        # 端口
        'PORT': 3306,
        # 数据库用户名
        'USER': 'root',
        # 数据库密码
        'PASSWORD': 'Lys19980705'
    }
}

    接下来需要指示Django使用pymysql模块连接mysql数据库，在与 settings.py 同级目录下的 init.py 中引入模块和进行配置

    import pymysql

pymysql.install_as_MySQLdb()

    # 三、模型定义

    # 3.1创建APP

    Django规定，如果要使用模型，必须要创建一个app，然后在项目目录下使用以下命令创建一个TestModel的app：django-admin startapp Test_App

    项目树如下所示：

    修改Test_App目录中的models.py文件

    # models.py
from django.db import models

class Test(models.Model):
    name = models.CharField(max_length=20)

    以上的类名代表了数据库表名，且继承了models.Model，类里面的字段代表数据表中的字段(name)，数据类型则由CharField（相当于varchar）、DateField（相当于datetime）， max_length 参数限定长度。

    然后在settings.py中找到INSTALLED_APPS这一项，如下：

    INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
  	# 新添加的App文件
    'Test_App'
]

    在命令行中运行（其中Test_App是新建的App名称）：

    $ python manage.py makemigrations
$ python manage.py migrate   # 创建表结构

$ python manage.py makemigrations Test_App  # 让 Django 知道我们在我们的模型有一些变更
$ python manage.py migrate Test_App   # 创建表结构

    • 问题：Django 输入python manage.py migrate不能新建表

    一般模型更改时执行一下两条指令：

    • python manage.py makemigrations

    • python manage.py migrate

    但是如果把两个表通过可视化工具给删掉了，再执行这两条语句就不好使了

    这是因为系统自动生成的有一张表django_migrations专门用来记录python manage.py makeigrations操作的，与项目根目录下migrations文件夹中的文件是一一对应的。

    通过可视化工具，将两条记录删掉，顺便把app下的migrations也给删掉。再次使用上述两条指令建表，就可以顺利建表

    参考：https://blog.csdn.net/hrbust_cxl/article/details/106763043

    # 3.2数据库操作

    # 3.2.1添加数据

    在settings.py同级包下创建一个testdb.py文件，如下所示

    from django.http import HttpResponse
from Test_App.models import Test


# 数据库操作
def testdb(request):
    test1 = Test(name='runoob')
    test1.save()
    return HttpResponse("<p>数据添加成功！</p>")

    同样的方式在urls.py中进行配置

    from django.urls import path
from . import views
from . import testdb

urlpatterns = [
    path('hello/', views.hello),
    path('test/', views.test),
    path('mysql/', testdb.testdb)
]

    访问该网址后发现数据库中添加了一个名为runoob的字段

    # 3.2.2查询数据

    在testdb.py文件中

    def testdb_quire(request):
    # 初始化
    response = ""
    response1 = ""

    # 通过objects这个模型管理器的all()获得所有数据行，相当于SQL中的SELECT * FROM
    q_list = Test.objects.all()

    # filter相当于SQL中的WHERE，可设置条件过滤结果
    response2 = Test.objects.filter(id=1)

    # 获取单个对象
    response3 = Test.objects.get(id=1)

    # 限制返回的数据 相当于 SQL 中的 OFFSET 0 LIMIT 2;
    Test.objects.order_by('name')[0:2]

    # 数据排序
    Test.objects.order_by("id")

    # 上面的方法可以连锁使用
    Test.objects.filter(name="runoob").order_by("id")

    # 输出所有数据
    for var in q_list:
        response1 += var.name + " "
    response = response1
    return HttpResponse("<p>" + response + "</p>")

    # 3.2.3修改数据

    在testdb.py文件中

    # 数据库更新
def testdb_edit(request):
    # 修改其中一个id=1的name字段，再save，相当于SQL中的UPDATE
    test1 = Test.objects.get(id=1)
    test1.name = 'Google'
    test1.save()

    # 另外一种方式
    # Test.objects.filter(id=1).update(name='Google')

    # 修改所有的列
    # Test.objects.all().update(name='Google')

    return HttpResponse("<p>修改成功</p>")

    # 3.2.4删除数据

    在testdb.py文件中

    # 数据库删除
def testdb_delete(request):
    # 删除id=1的数据
    test1 = Test.objects.get(id=1)
    test1.delete()

    # 另外一种方式
    # Test.objects.filter(id=1).delete()

    # 删除所有数据
    # Test.objects.all().delete()

    return HttpResponse("<p>删除成功</p>")
