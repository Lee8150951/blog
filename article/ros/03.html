    # 一、发布者Publisher编程实现

    # 1.1创建功能包

    首先在src目录下使用命令创建功能包

    catkin_create_pkg learning_topic roscpp rospy std_msgs geometry_msgs turtlesim

    项目结构：

    • /TestWorkspace是工作空间文件夹

    • /TestWorkspace/src是工作空间下功能包的存放位置

    • /TestWorkspace/src/learning_top表示一个功能包

    • /TestWorkspace/src/learning_top/src表示该功能包所有功能实现的源代码

    # 1.2编写功能

    完成功能包后开始编写功能代码

    /**
 * 该例程将发布turtle1/cmd_vel话题，消息类型geometry_msgs::Twist
 */
#include <ros/ros.h>
#include <geometry_msgs/Twist.h>
    
int main(int argc, char **argv)
{
	// ROS节点初始化
	ros::init(argc, argv, "velocity_publisher");
	// 创建节点句柄
	ros::NodeHandle n;
	// 创建一个Publisher，发布名为/turtle1/cmd_vel的topic，消息类型为geometry_msgs::Twist，队列长度10
	ros::Publisher turtle_vel_pub = n.advertise<geometry_msgs::Twist>("/turtle1/cmd_vel", 10);
	// 设置循环的频率
	ros::Rate loop_rate(10);
	int count = 0;
	while (ros::ok())
	{
	    // 初始化geometry_msgs::Twist类型的消息
		geometry_msgs::Twist vel_msg;
		vel_msg.linear.x = 0.5;
		vel_msg.angular.z = 0.2;
	    // 发布消息
		turtle_vel_pub.publish(vel_msg);
		ROS_INFO("Publsh turtle velocity command[%0.2f m/s, %0.2f rad/s]", 
				vel_msg.linear.x, vel_msg.angular.z);
	    // 按照循环频率延时
	    loop_rate.sleep();
	}
	return 0;
}

    # 1.2.1代码解读

    • 所有ROS节点都要做的一件事情就是ROS节点的初始化
    // ROS节点初始化
ros::init(argc, argv, "velocity_publisher");

    在节点初始化环节需要对节点进行命名，在ROS中节点名是不可以重复的

    该代码中argc和argv是调用该函数时传进来的两个参数并参与节点的初始化，通常情况下在初始化环节写节点名即可

    • 创建句柄
    // 创建节点句柄
ros::NodeHandle n;

    句柄主要使用与管理ROS相关API资源，即用于管理节点资源

    • 创建一个Publisher
    // 创建一个Publisher，发布名为/turtle1/cmd_vel的topic，消息类型为geometry_msgs::Twist，队列长度10
ros::Publisher turtle_vel_pub = n.advertise<geometry_msgs::Twist>("/turtle1/cmd_vel", 10);

    <geometry_msgs::Twist>是该消息的数据结构，尖括号内部里面填写的是要发布消息的数据类型

    ("/turtle1/cmd_vel", 10)是两个部分，第一个部分是发布的话题名是什么（必须和订阅的话题名相同，否则会出现管道不同的问题），第二个部分是队列长度

    • 设置频率
    // 设置循环的频率
ros::Rate loop_rate(10);

    这一块主要负责设置pub的频率，后面的代码会进入一个循环，也就是设定了循环的频率

    • 设置Twist消息的内容
     // 初始化geometry_msgs::Twist类型的消息
geometry_msgs::Twist vel_msg;
vel_msg.linear.x = 0.5;
vel_msg.angular.z = 0.2;

    首先创建类对象geometry_msgs::Twist vel_msg;该对象中包含了线速度和角速度：

    线速度：vel_msg.linear.x = 0.5;（m/s）

    角速度：vel_msg.angular.z = 0.2;（度/s）

    • 发布数据
    // 发布消息
turtle_vel_pub.publish(vel_msg);
ROS_INFO("Publsh turtle velocity command[%0.2f m/s, %0.2f rad/s]", 
		vel_msg.linear.x, vel_msg.angular.z);

    在前面创建的publisher：turtle_vel_pub.publish中有一个方法publish，该方法的参数就是输出的内容

    之后ROS会根据底层的通讯机制将数据压入消息队列中，队列会不断的发送出去

    ROS_INFO类似与C语言中的printf方法不断地在控制台进行日志输出，可以在客户端看到相关发送的信息

    # 1.2.2流程总结

    如何实现一个Publisher？

    • 初始化ROS节点
    • 向ROS Master注册节点信息，包括发布的话题名和话题中的消息类型
    • 创建消息数据
    • 按照一定频率循环发布消息

    # 1.3配置编译规则

    配置CMakeLists.txt的编译规则

    • 设置需要编译的代码和生成的可执行文件
    • 设置链接库

    在该文件build配置区域内添加以下代码

    add_executable(velocity_publisher src/velocity_publisher.cpp)
target_link_libraries(velocity_publisher ${catkin_LIBRARIES})

    add_executable(velocity_publisher src/velocity_publisher.cpp)是配置代码的编译

    target_link_libraries(velocity_publisher ${catkin_LIBRARIES})是配置可执行文件的链接

    # 二、订阅者Subscriber编程实现

    # 2.1编写功能

    在相应功能包中编写功能代码，如下：

    /**
 * 该例程将订阅/turtle1/pose话题，消息类型turtlesim::Pose
 */
#include <ros/ros.h>
#include "turtlesim/Pose.h"

// 接收到订阅的消息后，会进入消息回调函数
void poseCallback(const turtlesim::Pose::ConstPtr& msg)
{
    // 将接收到的消息打印出来
    ROS_INFO("Turtle pose: x:%0.6f, y:%0.6f", msg->x, msg->y);
}
int main(int argc, char **argv)
{
    // 初始化ROS节点
    ros::init(argc, argv, "pose_subscriber");
    // 创建节点句柄
    ros::NodeHandle n;
    // 创建一个Subscriber，订阅名为/turtle1/pose的topic，注册回调函数poseCallback
    ros::Subscriber pose_sub = n.subscribe("/turtle1/pose", 10, poseCallback);
    // 循环等待回调函数
    ros::spin();
    return 0;
}

    # 2.1.1代码解读

    • 初始化ROS节点

    同样的，所有的ROS节点都需要初始化

    • 创建句柄管理资源
    • 创建一个Subscriber
    // 创建一个Subscriber，订阅名为/turtle1/pose的topic，注册回调函数poseCallback
ros::Subscriber pose_sub = n.subscribe("/turtle1/pose", 10, poseCallback);

    对象的初始化信息包括：需要订阅的话题名"/turtle1/pose"（注意话题名一定要匹配），队列长度，回调函数

    回调函数的作用在于订阅者并不知道发布者的数据什么时候进入，一旦有数据进入，Subscriber就必须立马处理，也就是说这个消息并没有同步性

    • 循环等待
    // 循环等待回调函数
ros::spin();

    spin()函数是ROS系统中的一个死循环函数，进入该循环后会不断监听是否有数据进来，一旦有数据进入则会执行回调函数

    也就是说，程序在正常情况下不会运行到return 0;

    • 回调函数
    // 接收到订阅的消息后，会进入消息回调函数
void poseCallback(const turtlesim::Pose::ConstPtr& msg)
{
    // 将接收到的消息打印出来
    ROS_INFO("Turtle pose: x:%0.6f, y:%0.6f", msg->x, msg->y);
}

    回调函数的接受参数是针对消息的一个指针

    turtlesim::Pose是消息的结构，与话题发布的内容完全一致

    ConstPtr是一个长指针

    // 将接收到的消息打印出来
ROS_INFO("Turtle pose: x:%0.6f, y:%0.6f", msg->x, msg->y);

    ROS_INFO()用于打印实时位置

    # 2.1.2流程总结

    如何实现一个Subscriber？

    • 初始化ROS节点
    • 订阅需要的话题
    • 循环等待话题消息，接收到消息后进入回调函数
    • 在回调函数中完成信息处理

    注意：回调函数中对于数据的处理一定要高效，时间不能太长

    # 三、话题消息定义与使用

    在自定义开发中可能会面临ROS已经定义好的消息可能无法满足开发需求，需要自己来定义话题消息

    当然整个通讯还是在ROS Master管理器下进行的

    # 3.1自定义话题消息

    步骤总览：

    • 定义msg文件
    • 在package.xml中添加功能包依赖
    • 在CMakeLists.txt中添加编译选项
    • 编译生成语言相关文件

    # 1）定义msg文件

    自定义话题文件一般都放在了相应功能包下的msg文件夹中

    在msg文件夹下创建Person.msg文件并写入以下代码

    string name
uint8 sex
uint8 age

uint8 unknown = 0
uint8 male = 1
uint8 female = 2

    # 2）添加功能包依赖

    在当前功能包下的pakage.xml中添加依赖（固定配置）

    <build_depend>message_generation</build_depend>
<exec_depend>message_runtime</exec_depend>

    <build_depend>标签内部为编译依赖

    <exec_depend>标签内部为运行依赖

    # 3）添加编译选项

    • 由于添加了新功能包，所以需要在CMakeLists.txt文件中添加相应功能包查询选项（位于find_pakage栏目中）
    find_package(catkin REQUIRED COMPONENTS
  geometry_msgs
  roscpp
  rospy
  std_msgs
  turtlesim
  message_generation
)
    • 在CMakeLists.txt中配置msg编译设置（位于Declare ROS messages, services and actions栏目中）
    add_message_files(FILES Person.msg)
generate_messages(DEPENDENCIES std_msgs)
    • 添加运行时的依赖（位于catkin_package栏目中）
    catkin_package(
#  INCLUDE_DIRS include
#  LIBRARIES learning_topic
  CATKIN_DEPENDS geometry_msgs roscpp rospy std_msgs turtlesim message_runtime
#  DEPENDS system_lib
)
    • catkin_make指令进行编译

    编译成功后即可以在/devel/include下找到该.msg文件编译出来的.h文件

    # 3.2创建Publisher和Subscriber

    # 1）Publisher

    /**
 * 该例程将发布/person_info话题，自定义消息类型learning_topic::Person
 */
#include <ros/ros.h>
#include "learning_topic/Person.h"

int main(int argc, char **argv)
{
    // ROS节点初始化
    ros::init(argc, argv, "person_publisher");
    // 创建节点句柄
    ros::NodeHandle n;
    // 创建一个Publisher，发布名为/person_info的topic，消息类型为learning_topic::Person，队列长度10
    ros::Publisher person_info_pub = n.advertise<learning_topic::Person>("/person_info", 10);
    // 设置循环的频率
    ros::Rate loop_rate(1);
    int count = 0;
    while (ros::ok())
    {
        // 初始化learning_topic::Person类型的消息
    	learning_topic::Person person_msg;
		person_msg.name = "Tom";
		person_msg.age  = 18;
		person_msg.sex  = learning_topic::Person::male;
        // 发布消息
		person_info_pub.publish(person_msg);
       	ROS_INFO("Publish Person Info: name:%s  age:%d  sex:%d", 
				  person_msg.name.c_str(), person_msg.age, person_msg.sex);
        // 按照循环频率延时
        loop_rate.sleep();
    }
    return 0;
}

    值得注意的就是在发布消息时不再像发布ROS自带信息一般

    而是需要初始化Person类型的消息

    // 初始化learning_topic::Person类型的消息
learning_topic::Person person_msg;
person_msg.name = "Tom";
person_msg.age  = 18;
person_msg.sex  = learning_topic::Person::male;

    其他方式与标准方式类似

    注意：引入的头文件为编译出的.h文件

    # 2）Subscriber

    /**
 * 该例程将订阅/person_info话题，自定义消息类型learning_topic::Person
 */
#include <ros/ros.h>
#include "learning_topic/Person.h"

// 接收到订阅的消息后，会进入消息回调函数
void personInfoCallback(const learning_topic::Person::ConstPtr& msg)
{
    // 将接收到的消息打印出来
    ROS_INFO("Subcribe Person Info: name:%s  age:%d  sex:%d", 
			 msg->name.c_str(), msg->age, msg->sex);
}
int main(int argc, char **argv)
{
    // 初始化ROS节点
    ros::init(argc, argv, "person_subscriber");
    // 创建节点句柄
    ros::NodeHandle n;
    // 创建一个Subscriber，订阅名为/person_info的topic，注册回调函数personInfoCallback
    ros::Subscriber person_info_sub = n.subscribe("/person_info", 10, personInfoCallback);
    // 循环等待回调函数
    ros::spin();
    return 0;
}
