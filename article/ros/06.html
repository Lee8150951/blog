    ROS入门开发小结（Python篇）

    Oreo! 2021/8/4 ROSPython机器人开发

    # 一、发布/订阅模型

    # 1.1创建功能包

    $ catkin_create_pkg topic roscpp rospy std_msgs geometry_msgs turtlesim

    功能包解析：

    • roscpp：ROS系统的C++实现，提供了一个客户端库（client library），使C++开发能快速与ROS的话题、服务和参数配置进行交互

    • rospy：ROS系统的Python实现，提供一个Python的客户端库，其API使Python开发能快速与ROS话题、服务和参数配置进行交互

    • std_msgs：ROS系统中标准的消息类型库，包含原始数据类型和其他基本消息结构的常见消息类型（标准消息）

    • geometry_msgs：该功能包用于为常见的集合单元（例如：点、向量等）提供消息，促进整个系统互操作性（Twist消息）

    注意一：如果使用Python进行开发，需要在功能包下新建/scripts/文件夹，用于存放相应的Python源码文件

    注意二：在ROS中使用Python开发应注意要让Python文件有可执行权限

    注意三：由于python文件本身就是可执行文件，所以在使用python开发时不需要对源文件进行编译

    # 1.2Publisher实现

    #!/usr/bin/env python
# -*- coding: utf-8 -*-
import rospy
from geometry_msgs.msg import Twist

def velocity_publisher():
   # ROS节点初始化
    rospy.init_node('velocity_publisher', anonymous=True)

   # 创建一个Publisher，发布名为/turtle1/cmd_vel的topic，消息类型为geometry_msgs::Twist，队列长度10
    turtle_vel_pub = rospy.Publisher('/turtle1/cmd_vel', Twist, queue_size=10)

   #设置循环的频率
    rate = rospy.Rate(10) 

    while not rospy.is_shutdown():
      # 初始化geometry_msgs::Twist类型的消息
        vel_msg = Twist()
        vel_msg.linear.x = 0.5
        vel_msg.angular.z = 0.2

      # 发布消息
        turtle_vel_pub.publish(vel_msg)
       rospy.loginfo("Publsh turtle velocity command[%0.2f m/s, %0.2f rad/s]", 
            vel_msg.linear.x, vel_msg.angular.z)

      # 按照循环频率延时
        rate.sleep()

if __name__ == '__main__':
    try:
        velocity_publisher()
    except rospy.ROSInterruptException:
        pass

    在用Python实现ROS开发时尤其要注意一定要书写一下两行代码在文件的开头：

    #!/usr/bin/env python
# -*- coding: utf-8 -*-
    • #!/usr/bin/env python

    该语句实际上应该备份为两部分进行解释：#!和/usr/bin/env python

    #!被称作“释伴”（Shebang），其通常出现在类Unix系统的脚本中第一行，作为前两个字符。在#!之后，可以有一个或数个空白字符，后接解释器的绝对路径，用于指明执行这个脚本文件的解释器

    /usr/bin/env python即强调了对这段脚本的解释器是python解释器

    • # -*- coding: utf-8 -*-

    如果不添加此行代码会出现SyntaxError: Non-ASCII character '\xe5' in file错误

    原因Python默认以ASCII作为编码方式，如果在自己的Python源码中包含了中文等语言，此时将源文件以UTF-8格式保存了，仍然无法成功运行

    该段代码的意义就是解决此类问题的发生

    # 1.2.1代码解读

    1）初始化

    # ROS节点初始化
rospy.init_node('velocity_publisher', anonymous=True)

    初始化ROS节点，在python编程方式中调用rospy中的init_node()方法即可

    使用方法：rospy.init_node('nodeName', anonymous=True)

    传入两个参数，第一个参数指的是节点名，第二个参数anonymous表示后面定义相同的节点名时候，是否按照按照序号进行排列，传入True/False

    2）创建Publisher

    # 创建一个Publisher，发布名为/turtle1/cmd_vel的topic，消息类型为geometry_msgs::Twist，队列长度10
turtle_vel_pub = rospy.Publisher('/turtle1/cmd_vel', Twist, queue_size=10)

    使用rospy.Publisher()方法创建一个发布者，需要传入3个参数，分别是：话题名、消息类型、消息队列，标准格式如下：

    pub = rospy.Publisher('topic_name', std_msgs.msg.String, queue_size=10)

    3）设置循环频率

    #设置循环的频率
rate = rospy.Rate(10)

    调用rospy下的Rate方法，控制后续while循环的循环频率，单位为Hz

    4）循环部分

    while not rospy.is_shutdown():
    # 初始化geometry_msgs::Twist类型的消息
    vel_msg = Twist()
    vel_msg.linear.x = 0.5
    vel_msg.angular.z = 0.2

	# 发布消息
    turtle_vel_pub.publish(vel_msg)
    rospy.loginfo("Publsh turtle velocity command[%0.2f m/s, %0.2f rad/s]", 
	vel_msg.linear.x, vel_msg.angular.z)

	# 按照循环频率延时
	rate.sleep()

    此处是该节点的主循环，在节点为发生异常的情况下会一直在循环中运行，一旦发出异常ros.is_shutdown()会返回true值并跳出循环

    初始化一个geometry_msg信息，调用Twist()方法，其中linear表示线速度，angular表示角速度

    配置完消息后调用Publisher的.publish()方法发布消息，该方法需要传入一个消息作为参数

    发布消息后调用rospy.loginfo()方法打印日志信息至控制台

    循环内所有的操作完成后说明Publisher的一个工作周期已经完成，可以让节点休息一段时间，调用休眠函数.sleep()，节点进入休眠状态

    # 1.2.2流程总结

    • 初始化节点
    • 创建Publisher
    • 注册节点信息，包括话题名和话题内容
    • 按照频率循环发布消息（注意节点的休眠）

    # 1.3Subscriber实现

    #!/usr/bin/env python
# -*- coding: utf-8 -*-

import rospy
from turtlesim.msg import Pose

def poseCallback(msg):
    rospy.loginfo("Turtle pose: x:%0.6f, y:%0.6f", msg.x, msg.y)

def pose_subscriber():
	# ROS节点初始化
    rospy.init_node('pose_subscriber', anonymous=True)

	# 创建一个Subscriber，订阅名为/turtle1/pose的topic，注册回调函数poseCallback
    rospy.Subscriber("/turtle1/pose", Pose, poseCallback)

	# 循环等待回调函数
    rospy.spin()

if __name__ == '__main__':
    pose_subscriber()

    # 1.3.1代码解读

    1）初始化

    # ROS节点初始化
rospy.init_node('pose_subscriber', anonymous=True)

    2）创建Subscriber

    # 创建一个Subscriber，订阅名为/turtle1/pose的topic，注册回调函数poseCallback
rospy.Subscriber("/turtle1/pose", Pose, poseCallback)

    调用rospy.Subscriber()方法创建一个订阅者，该方法需要传入三个参数，分别是：订阅的话题名，数据类型，注册的回调函数

    3）回调函数

    def poseCallback(msg):
    rospy.loginfo("Turtle pose: x:%0.6f, y:%0.6f", msg.x, msg.y)

    回调函数接受一个消息参数，并通过loginfo方法进行控制台打印输出

    4）循环等待

    # 循环等待回调函数
rospy.spin()

    spin()方法的作用就是阻止python的退出，直到该节点被关闭

    # 1.3.2流程总结

    • 初始化ROS节点
    • 创建Subscriber，并订阅话题信息
    • 等待话题消息，接受到信息后进入回调函数
    • 在回调函数中对消息进行处理

    这是导入模块的另一种形式，使用这种import语句，调用模块中的函数时不需要moduleName.作为前缀。但是这种使用方式会导致代码的可读性降低

    # 2.2Service实现

    #!/usr/bin/env python
# -*- coding: utf-8 -*-
# 该例程将执行/turtle_command服务，服务数据类型std_srvs/Trigger

import rospy
import thread,time
from geometry_msgs.msg import Twist
from std_srvs.srv import Trigger, TriggerResponse

pubCommand = False;
turtle_vel_pub = rospy.Publisher('/turtle1/cmd_vel', Twist, queue_size=10)

def command_thread():	
	while True:
		if pubCommand:
			vel_msg = Twist()
			vel_msg.linear.x = 0.5
			vel_msg.angular.z = 0.2
			turtle_vel_pub.publish(vel_msg)
			
		time.sleep(0.1)

def commandCallback(req):
	global pubCommand
	pubCommand = bool(1-pubCommand)

	# 显示请求数据
	rospy.loginfo("Publish turtle velocity command![%d]", pubCommand)

	# 反馈数据
	return TriggerResponse(1, "Change turtle command state!")

def turtle_command_server():
	# ROS节点初始化
    rospy.init_node('turtle_command_server')

	# 创建一个名为/turtle_command的server，注册回调函数commandCallback
    s = rospy.Service('/turtle_command', Trigger, commandCallback)

	# 循环等待回调函数
    print "Ready to receive turtle command."

    thread.start_new_thread(command_thread, ())
    rospy.spin()

if __name__ == "__main__":
    turtle_command_server()

    # 2.2.1代码解读

    1）初始化

    # ROS节点初始化
rospy.init_node('turtle_command_server')

    2）创建Server

    # 创建一个名为/turtle_command的server，注册回调函数commandCallback
s = rospy.Service('/turtle_command', Trigger, commandCallback)

    调用Service()方法创建一个服务，需要传入三个值，分别是：服务名称，数据类型以及回调函数，然后开始循环等待服务请求，一旦有服务请求，Server就会跳入回调函数进行处理

    3）创建多线程

    thread.start_new_thread(command_thread, ())

    通过thread.start_new_thread()方法创建一个新的线程，其基本语法是：

    thread.start_new_thread ( function, (args[, kwargs]) )

    其中：

    • function表示对应的线程函数
    • args表示传递给线程函数的参数，必须是tuple元组类型

    线程的结束一般依靠线程函数的自然结束；也可以在线程函数中调用thread.exit()

    4）回调函数

    def commandCallback(req):
	global pubCommand
	pubCommand = bool(1-pubCommand)
	# 显示请求数据
	rospy.loginfo("Publish turtle velocity command![%d]", pubCommand)
	# 反馈数据
	return TriggerResponse(1, "Change turtle command state!")
    • python关键字global：global关键字允许在当前作用域之外修改变量，用于创建全局变量并在局部上下文中对该变量进行更改。
    • pubCommand = bool(1-pubCommand)：修改全局标志位

    5）线程函数

    def command_thread():	
	while True:
		if pubCommand:
			vel_msg = Twist()
			vel_msg.linear.x = 0.5
			vel_msg.angular.z = 0.2
			turtle_vel_pub.publish(vel_msg)
			
		time.sleep(0.1)

    线程函数实际上就是一个不断循环等待的函数，通过全局标志位来判定是否需要通过Publisher发布数据

    并使用time.sleep(s)方法让程序休眠，其中"s"以秒为单位，可以是小数，0.1秒则代表休眠100毫秒。

    # 2.2.2流程总结

    • 初始化ROS节点
    • 创建Server实例
    • 循环等待服务请求，进入回调函数
    • 在回调函数中完成服务功能并反馈应答数据

    # 2.3Client实现

    #!/usr/bin/env python
# -*- coding: utf-8 -*-
# 该例程将请求/spawn服务，服务数据类型turtlesim::Spawn

import sys
import rospy
from turtlesim.srv import Spawn

def turtle_spawn():
	# ROS节点初始化
    rospy.init_node('turtle_spawn')

	# 发现/spawn服务后，创建一个服务客户端，连接名为/spawn的service
    rospy.wait_for_service('/spawn')
    try:
        add_turtle = rospy.ServiceProxy('/spawn', Spawn)

		# 请求服务调用，输入请求数据
        response = add_turtle(2.0, 2.0, 0.0, "turtle2")
        return response.name
    except rospy.ServiceException, e:
        print "Service call failed: %s"%e

if __name__ == "__main__":
	#服务调用并显示调用结果
    print "Spwan turtle successfully [name:%s]" %(turtle_spawn())

    # 2.3.1代码解读

    1）初始化

    # ROS节点初始化
rospy.init_node('turtle_spawn')

    2）创建Client

    # 发现/spawn服务后，创建一个服务客户端，连接名为/spawn的service
rospy.wait_for_service('/spawn')
try:
    add_turtle = rospy.ServiceProxy('/spawn', Spawn)
    # 请求服务调用，输入请求数据
    response = add_turtle(2.0, 2.0, 0.0, "turtle2")
    return response.name
except rospy.ServiceException, e:
    print "Service call failed: %s"%e
    • 通过rospy.wait_for_service()方法，等待环境下是否存在名为/spawn的服务，如果存在则创建一个连接该服务的Client（这是一个阻塞性质的方法）
    • ServiceProxy类，rospy下的一个类，最主要的作用就是调用参数生成一个请求实例
    • ServiceException类，服务错误异常抛出

    # 2.3.2流程总结

    • 初始化ROS节点
    • 寻找服务，来创建一个Client实例
    • 请求服务调用，输入请求数据
    • 等待服务响应，反馈应答结果

    $ catkin_make
$ source devel/setup.bash
$ rosrun ***(功能包名) ***.py(脚本文件名)

    尤其要注意执行命令：source devel/setup.bash，否则会出现无法扫描到功能包的问题

    $ source filename

    source命令是bash shell的内置命令，其作用是在当前bash环境下读取并执行FileName中的命令
