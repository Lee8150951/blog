    VuePress简单搭建

    Oreo! 2021/3/19 VuePressvuepress-theme-reco

    # VuePress快速搭建

    官方网站：https://vuepress.vuejs.org/zh/

    # 项目生成

    1、在目录下构建根目录

    2、使用npm init初始化

    3、使用npm安装VuePress（新版的VuePress已经不再推荐使用全局安装）

    npm install -D vuepress

    4、在项目中建立docs文件夹（非常关键！），并写下第一篇文档README.md，实际上这篇文档就是整个项目的首页

    注意：文件名和文件夹名不要修改，保证为/docs/README.md

    5、通过脚本设定后启动项目npm run dev

    {
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }
}

    # 基本构建

    快捷构建的整个项目没有任何内容的存在，因为首页和配置文件都没有进行配置。

    下面对整个项目进行一个简单的配置

    目录结构：

    .
├── docs
│   ├── .vuepress
│   │   ├── theme (主题文件)
│   │   │   └── Layout.vue
│   │   ├── public (静态文件)
│   │   ├── styles (样式文件)
│   │   │   ├── index.styl
│   │   │   └── palette.styl
│   │   ├── config.js (主配置文件)
│   │ 
│   ├── README.md (首页)
│   ├── guide
│   │   └── README.md
│   └── config.md
│ 
└── package.json

    其中最为关键的就是config.js文件，这是整个项目的配置文件，先对其进行一个初步的修饰让整个初始构建的blog看起来更加得体

    以下展示整个配置文件的部分重要配置条目：

    module.exports = {
  // 网站的标题，它将会被用作所有页面标题的前缀，同时，默认主题下，它将显示在导航栏（navbar）上。
  title: 'VuePress',
  // 网站的描述，它将会以 <meta> 标签渲染到当前页面的 HTML 中
  description: 'Just playing around',
  // 额外的需要被注入到当前页面的 HTML <head> 中的标签，每个标签都可以以 [tagName, { attrName: attrValue }, innerHTML?] 的格式指定
  head: [
    ['link', { rel: 'icon', href: '/logo.png' }]
  ],
  // 指定用于 dev server 的主机名
  host: '0.0.0.0',
  // 指定 dev server 的端口
  port: '8080',
  // 指定客户端文件的临时目录
  temp: '/path/to/@vuepress/core/.temp',
  // 指定 vuepress build 的输出目录。如果传入的是相对路径，则会基于 process.cwd() 进行解析
  dest: '.vuepress/dist',
  // 提供多语言支持的语言配置。具体细节请查看 多语言支持
  locales: undefined,
  // 一个函数，用来控制对于哪些文件，是需要生成 <link rel="prefetch"> 资源提示的
  shouldPrefetch: () => true,
  // 用于指定 cache 的路径，同时也可以通过设置为 false 来在每次构建之前删除 cache
  cache: true,
  // 监听任何想监听的文件，文件变动将会触发 vuepress 重新构建，并实时更新
  extraWatchFiles: [
    '.vuepress/foo.js', // 使用相对路径
    '/path/to/bar.js'   // 使用绝对路径
  ],
  // 指定要解决的文件的模式
  patterns: ['**/*.md', '**/*.vue'],
  // 主题配置
  themeConfig: {
    // 增加导航栏 Logo ，Logo 可以被放置在公共文件目录
    themeConfig: {
      logo: '/assets/img/logo.png',
    },
    // 顶部导航栏
    nav: require('./nav'),
    // 侧边导航栏
    sidebar: 'auto',
    sidebarDepth: 2,
    smoothScroll: true
  }
}

    补充：为了使整个项目看起来更加简洁，建议使用module.exports和require()的ES6语法进行部分抽离，包括：nav.js、theme.js等。

    接下来对项目的首页进行简单构建，在新建的/docs/readme.md中进行配置

    ---
home: true
heroImage: /logo.png
heroText: Hero 标题
tagline: Hero 副标题
actionText: 快速上手 →
actionLink: /zh/guide/
features:
- title: 简洁至上
  details: 以 Markdown 为中心的项目结构，以最少的配置帮助你专注于写作。
- title: Vue驱动
  details: 享受 Vue + webpack 的开发体验，在 Markdown 中使用 Vue 组件，同时可以使用 Vue 来开发自定义主题。
- title: 高性能
  details: VuePress 为每个页面预渲染生成静态的 HTML，同时在页面被加载的时候，将作为 SPA 运行。
footer: MIT Licensed | Copyright © 2018-present Evan You
---

    # 导航栏

    此处导航栏指的是顶部右侧的链接按钮，这些按钮在没有配置相关主题的情况下都是起到的作用

    导航栏在config.js中进行配置

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    nav: [
      { text: 'Guide', link: '/guide/' }
    ]
  }
}

    在默认情况下：配置中的link虽然写的是文件夹，但是VuePress读取的是该文件夹下的README.md文件

    可以使用诸如：link: '/guide/about.md'，进行修改（这个情况下读的就是guide文件夹的about.md文件了）

    VuePress还提供了下拉式的导航栏菜单

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    nav: [
      {
        text: 'Languages',
        ariaLabel: 'Language Menu',
        items: [
          { text: 'Chinese', link: '/language/chinese/' },
          { text: 'Japanese', link: '/language/japanese/' }
        ]
      }
    ]
  }
}

    同样的，'/language/chinese'读的也是readme.md文件，其中/language/文件夹与前文/guide/文件夹同级

    此时项目的目录结构应为：

    .
├── docs
│   ├── .vuepress
│   │   ├── public (静态文件)
│   │   ├── styles (样式文件)
│   │   │   ├── index.styl
│   │   │   └── palette.styl
│   │   ├── config.js (主配置文件)
│   │ 
│   ├── README.md (首页)
│   ├── guide
│   │   └── about.md
│   ├── language
│   │   │   ├── chinese
│   │   │   │   └── README.md
│   │   │   └── japanese
│   │   │   │   └── README.md
│   └── config.md
│ 
└── package.json

    # 侧边栏

    配置根据md的目录自动展开（最方便）

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    sidebar: 'auto'
  }
}

    这就是整个项目的初始构建，下面将使用vuepress-theme-reco进行进一步美化构建

    # 使用vuepress-theme-reco

    官方网站：https://vuepress-theme-reco.recoluan.com/views/other/about.html

    # 基本引入

    1、安装：

    npm install vuepress-theme-reco --save-dev

    2、引用：

    // .vuepress/config.js
module.exports = {
  theme: 'reco'
}

    # 首页配置

    主题的主页的默认风格偏文档，并不像一个博客，vuepress-theme-reco的使用可以增加博客风格首页布局

    现在config.js中引入首页风格

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    type: 'blog'
  }
}

    对导航页的readme.md进行简单配置

    # 这是你的主页
---
home: true
heroImage: /hero.png
heroImageStyle: {
  maxHeight: '200px',
  display: block,
  margin: '6rem auto 1.5rem',
  borderRadius: '50%',
  boxShadow: '0 5px 18px rgba(0,0,0,0.2)'
}
---

    设置首页右侧信息栏头像

    // .vuepress/config.js
module.exports = {
  theme: 'reco',
  themeConfig: {
    authorAvatar: '/avatar.png'
  }
}

    # 导航栏

    总体方式和前面相同，但是值得注意的是在vuepress-theme-reco中新增了分类功能和标签功能，配置如下

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    blogConfig: {
      category: {
        location: 2,     // 在导航栏菜单中所占的位置，默认2
        text: 'fenlei' // 默认文案 “分类”
      },
      tag: {
        location: 3,     // 在导航栏菜单中所占的位置，默认3
        text: 'Tag'      // 默认文案 “标签”
      }
    }
  }
}

    这个功能的使用是配合每个文档前面的Front Matter使用的，标准的Front Matter语法如下（常用变量见官网）：

    ---
title: 烤鸭的做法
date: '2019-08-08 08:00:00'
sidebar: 'auto'
categories:
 - 烹饪
 - 爱好
tags:
 - 烤
 - 鸭子
keys:
 - '123456'
publish: false
---

    其中categories对应的是分类功能，tags功能对应的是标签功能

    # 语言配置

    将文件中的默认英文更改为默认中文

    // config.js
module.exports = {
  locales: {
    '/': {
      lang: 'zh-CN'
    }
  }
}

    # 侧边栏

    建议全局打开自动子侧边栏功能

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    subSidebar: 'auto'//在所有页面中启用自动生成子侧边栏，原 sidebar 仍然兼容
  }
}

    然后通过 YAML front matter 来禁用指定页面的侧边栏

    ---
subSidebar: false
---

    值得注意的是，使用vuepress-theme-reco对项目进行美化后，项目不必再强求使用readme.md方式读取，该主题会自动识别整个项目中的article，根据每个article不同的标签或分类分门别类，非常方便
