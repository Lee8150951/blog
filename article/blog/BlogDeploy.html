    VuePress部署到GitHub Pages

    Oreo! 2021/3/20 VuePressGitHub Pages

    # Git仓库创建

    在GitHub中创建一个仓库，命名随意，后文使用代替

    # 详细配置

    1、首先在/.vuepress/config.js中进行路径有关的配置

    // config.js
module.exports = {
  // 部署路径
  base: '/<REPO>/'
}

    2、在/docs/文件夹同级目录下创建部署脚本，命名为deploy.sh

    #!/usr/bin/env sh

# 抛出异常
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd ./.vuepress/dist

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -

    3、修改npm默认配置，通过npm run deploy直接运行脚本

    {
    "scripts": {
        "deploy": "deploy.sh"
    },
}

    注意：Windows系统下直接deploy.sh即可，在Linux系统下需要使用shell deploy.sh

    4、部署完成，此时在项目中会出现一个/.vuepress/dist/文件夹，里面存放的就是编译好的项目

    # 异常处理

    我在部署过程中出现了脚本闪退的问题，通过逐行排查发现问题就出在

    git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

    命令中，这条命令如果出现错误，比如push失败，cmd编译完成后是会直接关闭的。

    逐行排查的结果显示：git@github.com: Permission denied (publickey). Could not read from remote repository

    产生该问题的主要原因：

    Permission denied (publickey) 没有权限的publickey ，出现这错误一般是以下两种原因

    • 客户端与服务端未生成 ssh key
    • 客户端与服务端的ssh key不匹配

    解决办法我参考了https://blog.csdn.net/ywl470812087/article/details/104459288

    按步骤解决即可

    # 完成

    访问https://<USERNAME>.github.io/<REPO>/即可展示界面，如果出现404错误，再运行一次npm run deploy试试
