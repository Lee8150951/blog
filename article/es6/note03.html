    JavaScript中this指向问题

    Oreo! 2021/4/16 前端JavaScript

    # 概述

    定义：this的指向在函数定义的时候是确定不了的，只有函数执行的时候才能确定this到底指向谁，实际上this的最终指向的是那个调用它的对象

    回顾：

    • 所有浏览器都支持 window 对象，它表示浏览器窗口

    • 所有 JavaScript 全局对象、函数以及变量均自动成为 window 对象的成员

    • 全局变量是 window 对象的属性

    • 全局函数是 window 对象的方法

    # 指向情况分析

    # 全局方法中的this

    function a() {
    var user = "user"
    console.log(this.user)
    //undefined
    console.log(this)
    //Window
}
a()

    本例中a()作为直接定义在全局中的一个方法，直接使用a()进行调用时，调用它的对象是window，所以指向的为window，而window中没有user这个全局属性，所以显示undefined

    实际上以上代码中a()相当于window.a()直接表明了a方法的调用对象，this指向非常清楚

    # 对象方法中的this

    const o = {
    user: "user",
    fn: function(){
        console.log(this.user);  //user
    }
}
o.fn()

    和上面的理论一样，但是这是直接在对象中定义的方法，该方法中的this指向的就是定义它的对象o

    我的理解是：o对象的存在相当于给方法与window对象之间增加了一层外壳，使得this直接指向的是其o对象，而非window

    小结：

    • 如果一个函数中有this，但是它没有被上一级的对象所调用，那么this指向window，但在严格模式中this指向的不是

    • 如果一个函数中有this，这个函数有被上一级的对象所调用，那么this指向的就是上一级的对象

    • 如果一个函数中有this，这个函数中包含多个对象，尽管这个函数是被最外层的对象所调用，this指向的也是它上一级的对象

    const o = {
    a: 10,
    b: {
        // a:12,
        fn: function() {
            console.log(this.a); //undefined
        }
    }
}
o.b.fn()

    如上例中，a: 12这行代码处于注释状态，也就是说b对象中的并没有a这个属性，所以在直接调用时显示undefined

    但是，如果a: 12这行代码解开注释，那么调用时将显示结果12

    # 赋值调用后的this

    const o = {
    a: 10,
    b: {
        a: 12,
        fn: function(){
            console.log(this.a)
            console.log(this)
        }
    }
}
let j = o.b.fn
j() // undefined, window
o.b.fn() // 12, b

    this永远指向的是最后调用它的对象

    在本例中，虽然函数fn()是被对象b所引用，但是在将fn()赋值给变量j的时候并没有执行所以最终指向的是window

    # 构造函数中的this

    function Fn() {
    this.user = "user"
}
var a = new Fn()
console.log(a.user)
// user

    里之所以对象a可以点出函数Fn()里面的user是因为new关键字可以改变this的指向，将这个this指向对象a

    秉承JavaScript中万物皆为对象的原则，new关键字创建就是一个对象实例

    调用这个函数Fn()的是对象a，那么this指向的自然是对象a

    那么为什么对象a中会有user，因为已经复制了一份Fn()函数到对象a中，用了new关键字就等同于复制了一份

    # return与函数中的this

    1、返回对象

    function fn() {  
    this.user = 'user'
    return {} 
}
var a = new fn
console.log(a.user) // undefined

    2、返回函数

    function fn() {  
    this.user = 'user'
    return function() {}
}
var a = new fn
console.log(a.user) // undefined

    3、返回数值

    function fn() {  
    this.user = 'user'
    return 1
}
var a = new fn  
console.log(a.user) // user

    4、返回undefine

    function fn() {  
    this.user = 'user'
    return undefined
}
var a = new fn  
console.log(a.user) // user

    5、返回null（特殊情况）

    function fn() {  
    this.user = 'user'
    return null
}
var a = new fn
console.log(a.user) // user

    总结：如果返回值是一个对象，那么this指向的就是那个返回的对象，如果返回值不是一个对象那么this还是指向函数的实例。

    但是存在null这个特殊情况，虽然说null是对象的一种，但是null的this指向并不会因为返回null而受到任何影响，和返回undefined的效果是相同的

    # 延时函数中的this

    function Person() {
    this.age = 0
    setTimeout(function() {
        console.log(this)
    }, 3000);
}

var p = new Person() // 3秒后返回 window 对象

    对于延时函数内部的回调函数的this指向全局对象window

    # 箭头函数中的this

    由于箭头函数不绑定this， 它会捕获其所在（即定义的位置）上下文的this值， 作为自己的this值

    function Person(){
  this.age = 0;

  setInterval(() => {
    this.age++; // |this| 正确地指向 p 实例
  }, 1000);
}

var p = new Person();

    可以理解成：箭头函数相当于透明壳，其this可以突破这层壳而直接指向外层
