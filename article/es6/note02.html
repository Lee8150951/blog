    箭头函数

    Oreo! 2021/4/14 ES6前端

    # 概述

    箭头函数表达式，又称Lambda表达式，是ES6对于函数的一种语法拓展

    箭头函数本身非常简洁，没有自己的this，arguments，super或new.target

    箭头函数表达式更适用于那些本来需要匿名函数的地方，并且它不能用作构造函数

    # 基础语法

    function (x) {
    return x * x;
}

    以上函数的标准写法相当于：x => x * x

    (param1, param2, …, paramN) => { statements }
(param1, param2, …, paramN) => expression
//相当于：(param1, param2, …, paramN) => { return expression; }

// 当只有一个参数时，圆括号是可选的：
(singleParam) => { statements }
singleParam => { statements }

// 没有参数的函数应该写成一对圆括号。
() => { statements }

    如果函数体内没有其他语句，而是直接返回一个非常简单的普通结果，可以称略return

    但是还存在一种函数体内除了返回之外，还有其他一些语句的情况，此时不能省略return

    x => {
    if (x > 0) {
        return x * x;
    } else {
        return - x * x;
    }
}

    此外，箭头函数返回值为对象是一定要加上{}，外面包裹圆():

    x => ({ foo: x })

    如果参数不是一个，就需要用括号()括起来：

    // 两个参数
(x, y) => x * x + y * y

// 无参数
() => 3.14

// 可变参数
(x, y, ...rest) => {
    var i, sum = x + y;
    for (i=0; i<rest.length; i++) {
        sum += rest[i];
    }
    return sum;
}

    # this指向

    结论：箭头函数内部的this是词法作用域，由上下文确定

    在箭头函数出现之前，每一个新函数根据它是被如何调用的来定义这个函数的this

    • 如果是该函数是一个构造函数，this指针指向一个新的对象
    • 在严格模式下的函数调用下，this指向undefined
    • 如果是该函数是一个对象的方法，则它的this指针指向这个对象

    但是箭头函数不会创建自己的this,它只会从自己的作用域链的上一层继承this

    例如：

    function Person(){
  this.age = 0;

  setInterval(() => {
    this.age++; // |this| 正确地指向 p 实例
  }, 1000);
}

var p = new Person();

    在以上代码中，传递给setInterval的函数内的this与封闭函数中的this值相同

    可以理解成：箭头函数相当于透明壳，其this可以突破这层壳而直接指向外层

    详细的this指向问题会在另一篇文章中解释
